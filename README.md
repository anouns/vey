# VEY.AI — Персональная Операционная Система Знаний (полностью локальная)

**VEY.AI** — это мощное десктопное приложение, которое превращает **все ваши личные файлы** (исходный код, документы, PDF-книги, заметки, изображения, сканы) в единую интеллектуальную базу знаний с встроенным ИИ.  

Приложение работает **строго на вашем компьютере** — 100% локально. Никакие данные никогда не отправляются в облако, если вы сами не выберете облачную модель. Вы просто выбираете папку с проектом или книгами — и нейросеть мгновенно понимает всё содержимое, отвечает на вопросы, анализирует код, предлагает правки, выполняет команды терминала и сохраняет полный контекст между запусками.

Версия приложения: **2.0**  
Последнее обновление: март 2026

## Для чего нужна программа VEY.AI и какие задачи она решает

**Главная проблема современных ИИ** — они не знают ваши личные данные и часто «галлюцинируют».  
VEY.AI решает эту проблему с помощью технологии **RAG (Retrieval-Augmented Generation)**: сначала система ищет самые релевантные куски из ваших файлов, а потом генерирует ответ **строго на их основе**.  

### Почему это важно именно для вас?
- Полная приватность: всё остаётся на вашем диске.
- Точность ответов: ИИ не придумывает, а опирается только на ваши документы.
- Скорость работы без интернета (после первого запуска).
- Один интерфейс заменяет десятки инструментов: поиск, анализ кода, терминал, заметки.

### Конкретные задачи, которые решает VEY.AI (подробно):

1. **Семантический поиск по всему рабочему пространству**  
   Вы спрашиваете естественным языком, а система находит не по ключевым словам, а по смыслу.  
   Пример запроса: «Покажи все места, где я писал про архитектуру RAG в прошлом проекте».  
   Ответ: точные цитаты + ссылки на файлы и строки кода.

2. **Глубокий анализ и рефакторинг кода**  
   Объяснение любой функции, поиск потенциальных багов, генерация unit-тестов, обновление документации, предложение улучшений.  
   Каждое изменение файла требует вашего явного подтверждения (кнопки «Применить» / «Отклонить»).

3. **Работа с документами любых форматов**  
   PDF, DOCX, TXT, EPUB, Markdown, изображения.  
   Встроенный OCR (Tesseract) распознаёт текст на сканах и фотографиях.

4. **Встроенный терминал прямо внутри чата**  
   Любые shell-команды начинаются с `!`.  
   Примеры: `!npm install`, `!git status`, `!python script.py`, `!ls -la`.  
   Вывод команды появляется прямо в чате с цветовой подсветкой.

5. **Автоматическое сохранение и переиспользование рабочего пространства**  
   При каждом запуске индексация обновляется автоматически.  
   Контекст чата сохраняется между сессиями.

6. **Гибкий выбор моделей искусственного интеллекта**  
   - Полностью локальная модель Qwen 2.5 Coder 1.5B (рекомендуется).  
   - Подключение Ollama (любая ваша модель).  
   - Быстрый облачный Groq (только если нужен максимум скорости).

**Сравнение с альтернативами**  
- ChatGPT / Claude: не знают ваши файлы, отправляют данные в облако.  
- Local RAG-решения (AnythingLLM, PrivateGPT): сложны в установке, нет удобного десктопного интерфейса.  
- VS Code + Copilot: только код, нет работы с документами и терминалом в одном окне.  
VEY.AI объединяет всё в одном красивом и лёгком приложении.

## Архитектурные решения: общая структура приложения

### Почему выбран гибридный стек (React + Python + Rust)

Фронтенд на веб-технологиях даёт удобство и скорость разработки UI.  
Тяжёлая ML-логика вынесена в Python, потому что экосистема PyTorch и transformers позволяет быстро добавлять новые модели.  
Обёртка Tauri на Rust обеспечивает минимальный размер, низкое потребление памяти и максимальную безопасность.  
Такое разделение позволяет команде разрабатывать каждую часть независимо и быстро.

### Как Tauri интегрируется с Python-бэкендом

Rust-код в `src-tauri/src/lib.rs` запускает FastAPI-сервер в фоновом процессе при старте приложения.  
Общение происходит по HTTP (localhost). Это решение выбрано вместо сложных IPC, потому что HTTP проще отлаживать и масштабировать.

### Почему именно Tauri 2.0, а не Electron

Electron требует встраивания всего Chromium — размер бандла огромный, RAM высокий.  
Tauri использует системный WebView (Edge на Windows, WebKit на macOS).  
Результат: приложение запускается за 1–2 секунды и потребляет в 5–10 раз меньше ресурсов.

## Как работает RAG-пайплайн внутри VEY.AI (пошагово)

1. Пользователь выбирает папку → приложение сканирует все файлы (игнорируя папки из правил).  
2. Каждый файл разбивается на чанки по правилам из `WORKSPACE_RAG_RULES.md` (размер чанка, пересечение, приоритет кода).  
3. Чанки преобразуются в векторные эмбеддинги с помощью модели `all-MiniLM-L6-v2`.  
4. Эмбеддинги сохраняются в локальную векторную базу (в памяти + на диске для быстрого восстановления).  
5. При новом запросе пользователя: текст запроса → эмбеддинг → поиск топ-10 самых похожих чанков по косинусному сходству.  
6. Найденные чанки + системный промпт + запрос отправляются в выбранную LLM (Qwen или Ollama).  
7. LLM генерирует ответ строго на основе чанков — галлюцинации исключены.  
8. Если пользователь попросил изменения файлов — ответ парсится и предлагается для подтверждения.

Это пайплайн гарантирует 100% точность по вашим данным и полную приватность.

## Архитектурные решения: обработка документов и OCR

### Почему pypdf + Tesseract OCR

pypdf быстро извлекает текст из PDF и DOCX.  
Tesseract добавляет поддержку изображений и сканов.  
Выбор сделан потому, что обе библиотеки лёгкие, работают оффлайн и дают высокий процент распознавания русского языка.

### Как происходит чанкинг документов

Чанки создаются с пересечением 20% (чтобы контекст не терялся).  
Для кода чанки делаются по функциям/классам.  
Для книг — по главам или фиксированному размеру 512 токенов.  
Все правила хранятся в редактируемом файле `WORKSPACE_RAG_RULES.md`.

## Архитектурные решения: встроенный терминал в чате

### Как работает выполнение shell-команд

Команды, начинающиеся с `!`, перехватываются в `ai_service.py`.  
Они выполняются через `subprocess` в изолированной среде с явным подтверждением пользователя.  
Вывод возвращается в чат с цветовой подсветкой (ANSI → HTML).  
Это решение позволяет не переключаться между окнами и сохраняет весь контекст в одном интерфейсе.

Как запустить программу (самый простой способ — готовый .exe)

Откройте страницу релизов:
https://github.com/anouns/vey/releases
Выберите самый верхний релиз (Latest).
В блоке Assets скачайте файл:
vey-ai-setup.exe (или vey-ai-windows-x64.exe).
Дважды кликните по скачанному файлу.
Если Windows покажет предупреждение «Неизвестный издатель» — нажмите «Подробнее» → «Запустить в любом случае».
Подождите автоматической установки (от 3 до 15 минут в зависимости от скорости интернета):
Программа сама установит необходимое Python-окружение.
Скачает локальную модель Qwen 2.5 Coder 1.5B (~3 ГБ).
Создаст все папки и индекс.
На экране будет большой прогресс-бар и понятные сообщения на русском.

После завершения автоматически откроется главное окно VEY.AI.
Первый тест работы:
Нажмите кнопку «Выбрать рабочую папку».
Укажите любую папку (для теста можно выбрать books/).
Подождите окончания индексации (полоса прогресса вверху).
В чате напишите:
Привет! Расскажи, что ты умеешь
или
!whoami
Программа должна ответить мгновенно.


Всё готово! Теперь вы можете работать с любой своей папкой.

Системные требования (подробно)

Операционная система: Windows 10/11 (64-bit), macOS 12+, Ubuntu 22.04+
ОЗУ: минимум 8 ГБ (рекомендуется 16 ГБ и выше)
Свободное место на диске: минимум 5 ГБ (модель + индекс)
Процессор: любой современный (Intel/AMD 4+ ядер)
Интернет: нужен только при первом запуске для скачивания модели
GPU: не обязательно, но NVIDIA с CUDA ускорит работу в 2–3 раза


Настройка и кастомизация

Редактируйте WORKSPACE_RAG_RULES.md — меняйте размер чанков, приоритеты типов файлов, игнорируемые папки.
В настройках приложения можно переключать модели, менять температуру, максимальную длину ответа.
Тёмная/светлая тема переключается автоматически по системным настройкам.
Можно добавить свои промпты в ai_service.py для кастомного поведения.


Возможные проблемы и их решения (Troubleshooting)

Модель не скачивается → проверьте стабильность интернета, попробуйте запустить от имени администратора.
Высокая нагрузка на CPU → уменьшите количество чанков в WORKSPACE_RAG_RULES.md.
Не видно папку → запустите от имени администратора или проверьте права доступа.
Терминал не работает → убедитесь, что в настройках разрешены shell-команды.
Приложение не запускается → удалите папку ~/.vey-ai и попробуйте снова.
Ошибка PyTorch → установите версию с CPU: pip install torch --index-url https://download.pytorch.org/whl/cpu.

Если проблема осталась — создайте Issue на GitHub с описанием и скриншотом.

Как внести вклад в проект

Форкните репозиторий.
Создайте ветку feature/название-вашей-фичи.
Внесите изменения и добавьте тесты.
Откройте Pull Request с подробным описанием.

Будем рады любым улучшениям UI, производительности, новым моделям или документации!
